# Code of Conduct

Contributors are expected to keep project discussions respectful, focused, and
constructive.

Examples of expected behavior:

- use welcoming and inclusive language
- give and receive technical feedback professionally
- respect differing viewpoints and experience levels
- focus criticism on code, behavior, and project outcomes

Examples of unacceptable behavior:

- harassment, threats, or personal attacks
- discriminatory language or imagery
- publishing private information without permission
- sustained disruption of project discussions

Project maintainers may remove comments, close issues, or restrict participation
when needed to keep the project productive and safe.
